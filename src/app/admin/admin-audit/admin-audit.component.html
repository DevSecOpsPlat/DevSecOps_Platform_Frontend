Journal d'audit

Historique complet et chronologique de toutes les actions — connexions, créations de compte, activations, modifications admin, etc.

Contenu du journal d'audit

  • Connexions — chaque connexion réussie ou échouée, verrouillage après 3 échecs
  • Comptes — création / suppression admin, activation, envoi d'e-mail d'activation
  • Statut — activation ou désactivation d'un compte par l'admin
  • Identifiants — changement de mot de passe ou d'e-mail (utilisateur ou admin)

Ce journal est immutable : il conserve tout l'historique. Les filtres affichent le nombre d'entrées par type. Pour les incidents de sécurité à traiter, voir Alertes.

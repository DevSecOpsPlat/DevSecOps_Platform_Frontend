Alertes

Incidents hors norme — attaques, verrouillages, changements d'identifiants, accès refusés.

Détection en temps réel

  • Connexion échouée — chaque tentative (compte connu ou identifiant inconnu)
  • Force brute compte — 3 échecs consécutifs → verrouillage 15 min
  • Force brute IP — 5 échecs / minute depuis la même IP → blocage 30 min
  • Pic HTTP — trop de requêtes par IP (200/min global, 20/min sur login)
  • Honeypot & scan — accès à /.env, /wp-admin, /admin/secret…
  • Payload malveillant — motifs SQL / XSS dans les paramètres
  • User-Agent suspect — absent ou outil d'attaque (sqlmap, nikto…)
  • Takeover — mot de passe ou e-mail modifié par l'utilisateur
  • IP bloquées — persistées en base (blocked_ips), restaurées au redémarrage
  • Rate limiting — Bucket4j (token bucket) : global, login, API admin

Les connexions réussies, créations de compte et actions admin courantes sont dans le Journal d'audit.
